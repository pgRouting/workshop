2. WMS-server med GeoServer¶

Nu när vi har en pl/pgsql-wrapper ska vi göra den tillgänglig som ett WMS-lager med hjälp av GeoServer.

Installationen av GeoServer ligger utanför ramen för den här workshopen, men om du använder OSGeoLive för den här workshopen har du redan GeoServer installerat.

2.1. Anslut till administrationssidan¶

För att kunna skapa WMS-lagret måste vi ansluta till administrationsgränssnittet för GeoServer. På OSGeoLive Desktop öppnar du menyn Applications på skrivbordet och sedan Geospatial > Web Services > GeoServer > Start GeoServer.

När servern är igång öppnar du administrationssidan <http://localhost:8082/geoserver/web>`_ i din webbläsare, klickar på knappen Login och anger administratörsuppgifterna för GeoServer:

Användarnamn:

admin

Lösenord:

geoserver

2.2. Skapa lagret¶

Nu när du är inloggad som administratör kan du skapa WMS-lagret. I GeoServer-terminologi kommer du att skapa en SQL-vy (se den officiella dokumentationen <https://docs.geoserver.org/latest/en/user/data/database/sqlview.html>`_ för mer information).

Det första du ska göra är att skapa en ny Workspace för pgrouting: i den vänstra menyn på sidan, i avsnittet Data, klickar du på Workspaces och sedan på Add new workspace.

Fyll i formuläret med:

Namn:

pgrouting

Namnrymds-URI:

https://pgrouting.org

Tryck sedan på knappen ”Skicka”.

Nästa steg: skapa en ny Store kopplad till arbetsytan. Klicka på Stores i vänstermenyn och sedan på Add new Store. Här kan du välja vilken typ av datakälla som ska konfigureras. Välj PostGIS.

Fyll i formuläret med:

  • Grundläggande butiksinformation:

Arbetsområde:

pgrouting

Datakälla:

pgrouting

  • Parametrar för anslutning:

värd:

localhost

port:

5432

databas:

city_routing

schema:

public

användare:

user

lösenord:

user

Resten av värdena kan lämnas orörda.

Slutligen är nästa uppgift att skapa Lager. Klicka på menyn Lager och sedan på Lägg till en ny resurs. Välj den nyskapade arbetsytan och lagringsparet: pgrouting:pgrouting.

Klicka på Konfigurera ny SQL-vy inne i texten.

Namnge vyn pgrouting och fyll i SQL-uttalandet med:

SELECT ST_MakeLine(route.geom) FROM (
    SELECT geom FROM wrk_fromAtoB('vehicle_net', %x1%, %y1%, %x2%, %y2%
  ) ORDER BY seq) AS route

I SQL view parameters klickar du på Guess parameters from SQL; listan som visas representerar parametrarna från SQL ovan.

För varje parameter:

  • Ställ in standardvärdet till: 0

  • Ändra det reguljära valideringsuttrycket till ^-?[\d.]+$

Det reguljära uttrycket matchar bara siffror.

I listan Attributes:

  • Tryck på Uppdatera, ett attribut ska visas (kallas st_makeline)

  • Ändra typen till LineString och SRID för geometrikolumnen från -1 till 4326

Spara formuläret.

Slutligen måste vi ställa in resten av lagret.

Det enda du behöver göra på den här skärmen är att se till att koordinatreferenssystemet är korrekt: geometrierna i databasen är i EPSG:4326 men vi vill visa dem i EPSG:3857 eftersom OpenLayers-kartan där lagret ska läggas ut är i denna projektion.

Bläddra ner till avsnittet koordinatreferenssystem och ändra Declared SRS till EPSG:3857 och SRS-hantering till Reproject native to declared.

Klicka på länkarna Beräkna från data och Beräkna från ursprungliga gränser för att automatiskt ställa in skiktgränserna. Klicka på Spara längst ned på sidan för att skapa lagret.
