2.2. Mumbai database¶

pgRouting is installed as an extension. This requires:

  • Supported PostgreSQL version * Supported PostGIS version

These requirements are met on OSGeoLive. When the required software is installed, open a terminal window by pressing:code:ctrl-alt-t and follow the instructions. Information about installing OSGeoLive can be found in Chapter 3 of pgRouting Workshop

Note

If you don’t have pgRouting installed. You can find the installation procedure at here

2.2.1. Create Mumbai area database compatible with pgRouting¶

 1# Create the database
 2createdb mumbai
 3
 4# login as user "user"
 5psql mumbai << EOF
 6
 7
 8-- add PostGIS functions
 9CREATE EXTENSION postgis;
10
11-- add pgRouting functions
12CREATE EXTENSION pgrouting;
13
14-- creating schemas for data
15CREATE SCHEMA roads;
16CREATE SCHEMA buildings;
17CREATE EXTENSION hstore;
18
19
20EOF

2.2.2. Get the Mumbai Data¶

The pgRouting workshop will make use of OpenStreetMap data of an area in Mumbai City. The instructions for downloading the data are given below

2.2.2.1. Downloading Mumbai data from OpenStreetMap¶

1CITY="mumbai"
2BBOX="72.8263,19.1021,72.8379,19.1166"
3wget --progress=dot:mega -O "$CITY.osm" "http://www.overpass-api.de/api/xapi?*[bbox=][@meta]"

2.2.3. Upload Mumbai data to the database¶

The next step is to run osm2pgrouting converter, which is a command line tool that inserts the data in the database, “ready” to be used with pgRouting. Additional information about osm2pgrouting can be found here

For this stepi the follwing is used:

  • The mumbai_buildings.xml and mumbai_roads.xml configuration files for osm2pgrouting.

  • The ~/Desktop/workshop/mumbai.osm data from the previous step

  • The mumbai database.

  • The terminal window ctrl-alt-t.

2.2.3.1. Importing Mumbai Roads¶

 1osm2pgrouting \
 2    -f "mumbai.osm" \
 3    -c "../roads.xml" \
 4    --schema "roads" \
 5    -d mumbai \
 6    -U user \
 7    -W user \
 8    --prefix "roads_" \
 9    --addnodes \
10    --tags \
11    --clean

Note

Depending on the osm2pgrouting version -W password is needed
